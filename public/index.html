INTEGRATION SESSIONS

We prepared a multiple types of integration sessions/meetings like "GPP SP feature" include ppt with demo, data gathering, solution reviews and training (hands on) meeting. We have integration sessions for interfaces (all out of the box), channels (data trandformation and mapping), gateways and clearing, SOA services, data uploads, acknowledgements, integrity maintanence. We build a commong format for all sessions. It starts with feature overview and continues with scenarios duscussions related to this feature. Each session holds a list of documents (business and technical guides, formatted messages, request/response examples, validation schemas) and workflow questioner. Sessions lifecycle is as following: we present feature and give an usage examples. After that we review scenarios list, which utilises this feature in open disscussion meetings. At the end of the session we convert scenarios questioner to basic integration setup document (out of the box rules and profile setup). Such document can be used as a entry point to extended sessions or refence guide for the COTS setup stage. We provide a two weeks support for each feature setup with extention and bank of 24 hours production support.
